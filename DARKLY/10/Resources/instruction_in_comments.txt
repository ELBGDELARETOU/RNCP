explication:
Ici, en inspectant la page link tout en bas, on se rend compte que des commentaires ont été laissés.
En suivant ce qui est demandé, on construit cette requête :

curl 'http://localhost:8080/index.php?page=b7e44c7a40c5f80139f0a50f3650fb2bd8d00b0d24667c4c2ca32c88e13b758f' \
  --compressed \
  -H 'User-Agent: ft_bornToSec' \
  -H 'Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8' \
  -H 'Accept-Language: en-US,en;q=0.5' \
  -H 'Accept-Encoding: gzip, deflate, br, zstd' \
  -H 'Referer: https://www.nsa.gov/' \
  -H 'Connection: keep-alive' \
  -H 'Cookie: I_am_admin=68934a3e9455fa72420237eb05902327' \
  -H 'Upgrade-Insecure-Requests: 1' \
  -H 'Sec-Fetch-Dest: document' \
  -H 'Sec-Fetch-Mode: navigate' \
  -H 'Sec-Fetch-Site: same-origin' \
  -H 'Sec-Fetch-User: ?1' \
  -H 'Priority: u=0, i' \
  -H 'Pragma: no-cache' \
  -H 'Cache-Control: no-cache' | grep flag 

On remplace le User-Agent par ft_bornToSec, car dans les commentaires on nous demande d’utiliser ce “browser”.
Ensuite, on fait la même chose pour www.nsa.gov où l’on nous dit que nous venons de ce site, ce qui fait référence au header Referer.

solution :
– ne pas laisser de commentaires qui peuvent aider à quoi que ce soit dans le code lisible par tout le monde
